Tipos de Testes 

-> Quando Testar

* Niveis de Teste

Teste de Unidade
-> Testa uma pequena parte do sistema
-> Geralmente automatizado
-> Subrotinas, metodos classes
-> Baixo nivel

Teste de Integração
-> Componentes do Sistema
-> Validar comunicações (envio e recebimento de info)
-> Antes do sistema ser concluido 
-> Fase intermediaria

Teste de Sistema
-> Testar como fosse um Usuario FInal
-> Equipe de Teste
-> Após a codificação do sistema 

Teste de Aceitação
-> Executado a nivel de sistema
-> Testado pelo Browser
-> Usuario que testará
-> Cliente usa o software e diz se aceita ou não 

Teste Alfa 
-> Não planejada
-> Executar o sistema para um grupo pequeno de pessoas
-> Geralmente membros da propria organização e cliente
-> Representantes acompanharam de perto os testes 

Teste de Aceitação x Teste Alfa
1 ou 2 clientes x Um grupo de clientes e membros da organiz~ção 

Teste Beta 
-> Não planjeado
-> Grande numero de pessoas desconhecidas com publico alvo
-> Não tem representantes acompanhando de perto 

Teste de Regressão
-> Quando ocorrer alguma alteração no sistema

° Como Testar?

Caixa Branca
-> Visualiza o Código do Sistema

	Analise Estatica
	-> Não Executa o código
	-> Analise de Exceções
	-> Boas Praticas não sendo aplicadas

Caixa Preta
-> Não preciso ver o Código do Sistema
-> Integração, Sistema, Aceitação, Alfa e Beta

Testes Manuais
->Lentos
->Não exige tecnologias
->Alto custo a cada execução
->Possue limitações com grande paralelismo
->Avaliar questões visuais e usabilidade

Testes Automaticos
->Rapidos
->Exige tecnologias especificas
->Alto custo apenas na criação
->Permite situações impossiveis manualmente
->Não avaliam questões visuais e usabilidade

Teste Estático
-> ex: Findbugs
-> Legibildidade, Nomenclaturas, Boas praticas
-> Apenas olhando o código

Teste Dinamico
-> Validar sistem atraves da execução
-> Insere entrada e confere saida

° Tipos de Testes

Teste de Funcionalidade
-> Se esta tudo funcionando corretamente
-> Envolve: Todos os Niveis

Teste de Desempenho
-> Tempo de resposta
-> ex: JMeter

Teste de Usabilidade
-> Validam aspectos do usuario
-> Reações 

Teste de Segurança 
-> Tentar entrar no sistema

Teste de Portabilidade
-> Funcionamento do Sistema em diversos dispositivos

Teste de Stress
-> Testar o sistema em condições extremas
-> ex: JMeter
-> Simular acesso de muitos usuarios ao mesmo tempo
